# Evidence Table

| Authors & Year | Objective | Dataset | Metrics | Key Findings | Limitations |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Siddiq et al., 2025** | Analyze the reproducibility crisis in LLM-for-SE research. | 640 SE/ML papers | Artifact availability rate | 43% of papers lack sufficient environment or prompt details for reproducibility. | Evaluates research papers' reproducibility, not direct LLM bug reproduction. |
| **Plein et al., 2023** | Investigate the feasibility of LLMs generating test cases from bug reports. | Bug report datasets | Feasibility, Executability | LLMs can translate natural language bug reports into executable test cases. | Focuses on generating test cases rather than evaluating report quality. |
| **Chaves et al., 2025** | Automate bug report generation using LLMs (T5) based on short tester prompts. | 5 real-world bug types | Acceptance Rate, Validity | >87.5% of generated reports were rated Valid/Partially Valid. Significantly reduces reporting time. | Still requires Human-in-the-loop to verify generated steps and prevent hallucinations. |
| **Zhang et al., 2024** | Critically review LLMs in SE, focusing on Automated Program Repair (APR). | 151 bugs (EVALGPTFIX) | Success rate | ChatGPT shows impressive repair performance but carries high risks of data leakage. | Emphasizes code repair over bug report triage. |
| **Kang et al., 2022** | Propose LIBRO framework to automatically reproduce bugs by generating tests via LLMs. | Defects4J | Precision, Reproducibility rate | LIBRO successfully reproduced 33% of bugs and can automatically select tests with high precision. | Performance heavily relies on the underlying LLM's context window. |
| **Kang et al., 2023** | Evaluate 15 diverse LLMs on automatic and general bug reproduction. | Defects4J, GHRB | Reproducibility rate | Larger models perform better; open-source models approach closed-source model performance. | Requires substantial computational resources for open-source model evaluation. |
| **Feng et al., 2024** | Propose AdbGPT to automate Android bug replay from natural language bug reports. | 88 bug reports | Accuracy, Reproducibility | Achieves >81% accuracy in bug replay, 5x faster than baseline methods. | Heavily dependent on the clarity of the user's initial bug description. |
| **Shi et al., 2025** | Propose a Human-in-the-Loop framework to improve LLM-as-a-Judge patch evaluation. | 115 patches | Cohen's Kappa, Precision, Recall | LLM-Judge achieves substantial agreement (Kappa 0.75) with humans when guided by clear rubrics. | LLMs still struggle with highly controversial patches (false positives). |
| **Albalawi et al., 2025** | Evaluate LLMs for extracting structured usability issues from user bug reports. | 370 user reviews | Precision, Recall, F1 | GPT-4 achieves the highest accuracy in classifying bug types and UI components. | Limited to mobile UI/usability bugs. |
| **Rasheed et al., 2024** | Propose an AI-agent using LLMs for autonomous code review and bug detection. | Code repos, Bug reports | Effectiveness | AI agents provide meaningful optimizations and proactively reduce post-release bugs. | Early results; requires broader quantitative empirical validation. |
