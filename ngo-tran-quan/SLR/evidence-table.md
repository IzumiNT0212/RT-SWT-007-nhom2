## Main evidence toward GAP

1. Many studies already use LLMs for bug report generation, improvement, reproduction, summarization, classification, categorization, and crash-report enhancement.
2. Several studies use quality-related metrics such as CTQRS, SBERT, ROUGE, semantic similarity, executable S2R rate, reproduction success rate, Top-1 localization, CodeBLEU, hallucination rate, and F1-score.
3. Some studies include human evaluation or annotation agreement, but they do not directly validate **LLM-based bug report quality assessment** against **developer manual review** using **Cohen’s Kappa ≥ 0.70** as the main success criterion.
4. Therefore, the confirmed GAP is: existing LLM-based bug report studies do not sufficiently answer whether an LLM can assess bug report reproducibility quality with substantial agreement compared to developer manual review.
